# Getting Started ‚Äî Build your first form in 5 minutes

This guide walks you from a fresh checkout to a working, GPU-rendered form running in the browser.

## Prerequisites

| Tool | Version | Install |
|------|---------|---------|
| Rust (stable) | 1.75+ | `curl https://sh.rustup.rs -sSf \| sh` |
| wasm-pack | latest | `cargo install wasm-pack` |
| Python 3 (or any static server) | any | comes with most OS |

No Node.js, no npm, no bundler required.

## 1. Clone and build

```bash
git clone https://github.com/your-org/wham.git
cd wham

# Build the WASM package (takes ~30 s on first run)
cd crates/ui-wasm
wasm-pack build --target web --out-dir ../../examples/web/pkg
cd ../..
```

## 2. Serve the demo

```bash
cd examples/web
python3 -m http.server 8080
```

Open `http://localhost:8080` in a browser that supports WebGL2 (Chrome, Firefox, Safari 15+, Edge).

You should see a demo form rendered entirely on a `<canvas>` ‚Äî no DOM inputs.

## 3. Anatomy of the demo app

```
examples/web/
  index.html     ‚Äî <canvas id="app"> + imports app.js as an ES module
  app.js         ‚Äî event wiring, AccessibilityMirror, rAF loop
  pkg/           ‚Äî compiled WASM + JS bindings (generated by wasm-pack)
```

The JavaScript in `app.js` does three things each frame:
1. Collects input events (pointer, keyboard, IME, wheel) into an array.
2. Calls `app.frame(events, width, height, dpr, timeMs)`.
3. Uses the returned accessibility JSON to update the hidden screen-reader DOM.

The Rust side (`ui-wasm`) calls your `build()` function each frame, which calls `ui-core` widget methods to produce a batch of GPU quads and text runs. The WebGL2 renderer uploads those quads and draws them.

## 4. Add a new text field step by step

Open `crates/ui-wasm/src/runtime.rs` and find the `build` function (or wherever widgets are emitted). Add a `TextBuffer` to your app state and call `ui.text_input`:

### Step 1 ‚Äî Declare the buffer in your app state

```rust
// In your application struct:
pub struct MyApp {
    name_buf: TextBuffer,
    // ...
}

impl MyApp {
    pub fn new() -> Self {
        Self {
            name_buf: TextBuffer::new(""),
            // ...
        }
    }
}
```

### Step 2 ‚Äî Emit the widget in `build`

```rust
fn build(&mut self, ui: &mut Ui, form: &mut Form) {
    ui.label("Full name");
    ui.text_input("name", &mut self.name_buf, "Enter your name");

    // Read the current value:
    let current_name = self.name_buf.text();
}
```

### Step 3 ‚Äî Or use the form-integrated auto-binding variant

If your field is already declared in the `FormSchema`, use `text_input_for` to skip manual `TextBuffer` management:

```rust
let path = FormPath::root().push("name");
ui.text_input_for(form, &path, "Full name", "Enter your name");
```

The buffer is created and synced automatically.

### Step 4 ‚Äî Re-build the WASM and reload

```bash
cd crates/ui-wasm
wasm-pack build --target web --out-dir ../../examples/web/pkg
```

Refresh the browser tab ‚Äî your new field should appear.

## 5. Adding the field to the schema

Define what data the form collects and how it validates:

```rust
fn schema(&self) -> FormSchema {
    FormSchema::new("registration")
        .field("name", FieldType::Text)
        .with_label("name", "Full name")
        .required("name")
        .field("email", FieldType::Text)
        .with_label("email", "Email address")
        .required("email")
        .with_validation("email", ValidationRule::Email)
}
```

The schema drives:
- Initial field values (all start empty/false/0).
- Validation ‚Äî call `form.validate()` before submission.
- Serialisation ‚Äî `form.start_submit(payload, retries)` validates and snapshots state.

## 6. Run the tests

```bash
# Unit + property tests for the Rust core
cargo test -p ui-core

# Browser integration tests (requires Chromium in PATH)
cargo test -p cdp-runner
```

Tests live alongside the code they test in `#[cfg(test)] mod tests` blocks.
